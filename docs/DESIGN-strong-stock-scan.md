# 产品设计：强势股归类扫描

> 发现"哪个自选分组正在批量冒出强势股"。把符合筛选条件（如涨幅 7%~12%）的股票按自选分组归类统计，看哪个分组的命中数最多，辅助判断当下主线。

## 一、目标与价值

盘中实时监控时，光看"哪个板块平均涨幅最高"还不够——板块平均会被大量没动的成分股稀释。真正有价值的信号是：**哪个分组里批量出现了强势股（如涨幅 7%~12% 的加速股）**。

例如：AI国产芯片分组 26 只里有 8 只涨 7%~12%（命中率 31%），而 PCB 分组 40 只里只有 2 只命中（5%）——说明当下主线在 AI 芯片，不在 PCB。这种"命中率"信号比"平均涨幅"更能反映资金集中度。

## 二、核心概念澄清

| 概念 | 定义 | 注意 |
|---|---|---|
| **筛选（scan）** | 按条件（涨幅范围/成交额/实体涨幅）过滤股票池 | 交互式，实时调参 |
| **归类（group-by）** | 把命中的股票按自选分组聚合统计 | 一股属多组，各组各计 +1 |
| **命中（hit）** | 某股票满足筛选条件 | 命中后计入它所属的所有分组 |
| **不是"概念归因"** | 本功能**不做** l1_stock_attribution（个股涨幅拆解到概念贡献） | 那是另一套逻辑，本功能是分组计数排名 |

### 交互

1. **筛选条件输入框**：涨幅区间（必填）、成交额下限（可选）、实体涨幅下限（可选）。点"应用筛选"或回车触发查询。
2. **分组命中统计表**：默认按命中数降序。列：排名 / 分组名 / 命中数 / 命中股平均涨幅 / 覆盖率（命中数/该分组成员总数）。
3. **手风琴展开**：点击分组行 → 下方展开该分组的命中个股表格（代码/名称/涨幅/涨速/实体/成交额/综合分）。**可同时展开多个**，再点折叠。
4. **时间条复用**：拖动/播放时间条时，按当时切片重新筛选归类（看"9:50 时哪个分组命中最多"）。
5. **3s 轮询**：自动跟随时，实时刷新命中结果。

## 四、数据基础

### 复用现有分时序列

完全复用 `realtime_engine` 的分时序列缓存（`custom_group` 缓存键），**不重新拉数据**。筛选归类只是对缓存里的切片做内存计算。

### 需补的字段（本次顺手做掉）

当前 `_build_indicator_df` 只算了 change_ratio/speed/body/acceleration，**缺成交额和实体涨幅**。本次补上：

| 字段 | 来源 | 算法 |
|---|---|---|
| `change_ratio` 涨幅 | 现有 | `(last - pre_close) / pre_close × 100` |
| `body` 实体涨幅 | 现有 | `(last - 开盘价) / 开盘价 × 100`（开盘至今） |
| `speed` 涨速 | 现有 | 1min 滚动末点 |
| `acceleration` 加速 | 现有 | 涨速差分 |
| **`amount` 成交额** | **新增** | 切片末点的累计 `turnover`（元，原始分时已含） |
| **`body_pct`** | **复用 body** | "实体涨幅"筛选维度 = body（开盘至今涨幅） |

> **"实体涨幅"语义**：分时数据每分钟点只有 `last_price` + 累计 `avg_price`，**没有单根 1min K 线的 OHLC**，算不出传统意义的"单根实体（close-open）"。故"实体涨幅"筛选维度采用 **body（开盘至今涨幅）** 语义，与现有看板一致。若未来需要"最近1分钟实体"，可用相邻 last_price 差近似。

## 五、后端接口

### `GET /api/custom/scan`

按筛选条件扫描，返回分组命中统计 + 命中个股明细。

**入参**：
| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `trade_date` | str | 今天 | YYYYMMDD |
| `snapshot_time` | str | 最新 | HH:MM（时间条切片） |
| `min_change` | float | 7 | 涨幅下限 % |
| `max_change` | float | 12 | 涨幅上限 % |
| `min_amount` | float | 空 | 成交额下限（万元），空=不限 |
| `min_body` | float | 空 | 实体涨幅下限 %，空=不限 |

**出参**：
```json
{
  "trade_date": "20260618",
  "snapshot_time": "10:30",
  "available_times": ["09:30", "09:31", ...],
  "pool_size": 594,
  "hit_total": 23,
  "group_hit_count": 18,
  "groups": [
    {
      "group_id": "236",
      "group_name": "AI国产芯片",
      "hit_count": 8,
      "member_total": 26,
      "coverage": 0.31,
      "hit_avg_change": 9.2,
      "hits": [
        {"code":"688xxx","name":"某芯片","change_ratio":9.5,"speed":1.2,"body":7.0,"amount":320000000,"score":1.8},
        ...
      ]
    },
    ...
  ]
}
```

### 实现要点

1. **数据源**：复用 `compute_dashboard(custom_mode=True)` 的分时切片（`_ensure_series` + `_build_indicator_df`），拿到所有股票的指标 DataFrame。
2. **筛选**：在 DataFrame 上按 `min_change ≤ change_ratio ≤ max_change`（+ 可选 amount/body）过滤，得命中股票集合。
3. **归类**：遍历 `custom_group` 的 `{group_id: [codes]}`，对每个分组取与命中集合的交集，得该分组命中明细。**一股属多组，各组的 hits 里都会出现它**。
4. **统计**：每分组算 hit_count / coverage（命中数/成员总数）/ hit_avg_change。按 hit_count 降序。

## 六、前端实现

### 新 Tab（`tabs.html` 加第三个）

### 新模板逻辑（`index.html` 加 `board=scan` 分支）

- 渲染逻辑换为：筛选条件表单 + 分组命中手风琴表

### 关键交互

- **筛选表单**：3 个 input（涨幅区间、成交额、实体涨幅），"应用筛选"按钮触发 `refresh()`，把条件拼进 URL 参数

- **多组同时展开**：支持，再点折叠

## 七、决策记录（已和用户对齐）

| 决策点 | 结论 |
|---|---|
| "归因"含义 | 按分组归类统计（非个股概念拆解） |
| 归类维度 | 仅自选分组（概念板块为未来扩展） |
| 筛选方式 | 前端交互式（非配置文件） |
| 多分组计数 | 一股属多组，各组各计 +1 |
| 默认筛选 | 涨幅 7%~12%，成交额/实体涨幅留空 |
| 展开模式 | 手风琴，可同时展开多个 |
| 实体涨幅语义 | = body（开盘至今涨幅），分时无单根 OHLC |
| 放置位置 | 顶层第三个 Tab |
| 时间功能 | 复用（时间条/播放/历史回看/3s轮询） |
